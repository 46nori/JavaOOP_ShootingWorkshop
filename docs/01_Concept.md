# Step 1: クラスの概念を理解しよう

コードを書く前に、まずは「クラス」とは何かを整理しましょう。Javaではすべてのプログラムがクラスの中に書かれます。

## 👾 敵キャラクターを作るには何が必要？

画面に敵キャラクターを表示して動かすためには、コンピュータにどんな情報を与えればよいでしょうか？
少し想像してみましょう。

### 1. 敵の状態（データ）

敵が「今どこにいるのか」「どっちに向かっているのか」を知る必要があります。

- **座標**: 画面上の位置（X座標, Y座標）
- **速度**: どれくらいの速さで動くか
- **方向**: どの方向に進むか

### 2. 敵の動き（操作）

データがあるだけでは動きません。そのデータを使って何をするかが必要です。

- **移動する**: 速度と方向に従って、座標を更新する
- **描画する**: 現在の座標に画像や図形を表示する

---

## 🤔 これをプログラムでどう管理する？

上記のような「データ」と「動き」を、プログラムの中でどう扱えばよいでしょうか？

### 方法A：クラスを使わない場合（大変！）

もしクラスを使わずに、たくさんの敵キャラを管理しようとすると、変数がバラバラになってしまいます。

```java
// 敵Aのデータ
double enemyA_x = 100;
double enemyA_y = 100;
double enemyA_speed = 5;

// 敵Bのデータ
double enemyB_x = 200;
double enemyB_y = 200;
double enemyB_speed = 3;

// 敵Cのデータ...（もう管理しきれない！）
```

これでは、敵が100体になったら変数が数百個も必要になってしまいます。また、「敵Aを動かす」処理と「敵Bを動かす」処理も別々に書かなければなりません。

### 方法B：クラスを使う場合（スッキリ！）

クラスを使うと、これら関連するデータ（座標、速度など）と処理（移動など）をひとまとめにして、「敵」という**新しい型**を作ることができます。

```java
// Enemyという型を作れば、スッキリ管理できる！
Enemy enemyA = new Enemy();
Enemy enemyB = new Enemy();
```

クラスを作ることは、**「整理整頓のための専用ボックス」** を作るようなものです。
このボックスの中に「データ」と「動き」を詰め込んでおくことで、プログラム全体が驚くほどシンプルになります。

## 重要なキーワード

- **クラス (Class)**: 設計図（型）。「敵キャラとはこういうもの」という定義です。
- **オブジェクト (Object) / インスタンス (Instance)**: 設計図から作られた実体。実際に画面に現れる1体1体の敵です。
- **フィールド (Field)**: オブジェクトが持つデータ。敵の「位置」や「体力」などです。
- **メソッド (Method)**: オブジェクトが行う動作。「動く」「攻撃する」などの命令です。

## 設計: Enemyクラス

今回は、敵キャラクターを表す `Enemy` クラスを次のように設計してみましょう。

| 種類 | 名前 | 型 | 説明 |
| --- | --- | --- | --- |
| **フィールド (Field)** | `x`, `y` | `double` | 現在の座標 |
| | `angle` | `double` | 移動方向（角度） |
| | `speed` | `double` | 移動速度 |
| **メソッド (Method)** | `Enemy` | (Constructor) | 初期化（コンストラクタ） |
| | `draw` | `void` | 画面に自分を描画する |
| | `erase` | `void` | 画面から自分を消す |
| | `move` | `void` | 座標を更新する |
